### 🔴 CRITICAL（3个）

**Spec 1: C1-章节被修订元输出覆盖**
- slug: `fix-revision-overwrite-chapter-content`
- 根因：`shenbi-chapter-revision` 在 no-op/auto-skip 路由下将修订摘要写入 `chapters/chapter-N.md` 而非保留原正文

- 修复方向：修订 skill 增加 content-preservation 守卫；dispatch_helper 对修订跳过场景保留原文件

**Spec 2: C2-文风渐进式崩塌**
- slug: `fix-progressive-prose-collapse`
- 根因：Ch30起系统术语密度从2‰升至135‰（Ch53）— `shenbi-drift-guidance` 的单调下降检测（`compute_drift.py:86-107`）可能因 resonance 评分未显著下降而延迟触发
- 关键文件：`skills/shenbi-drift-guidance/SKILL.md`, `src/shenbi/skill_utils/drift_detection/compute_drift.py:86-146`
- 修复方向：增加"系统术语密度"作为独立漂移指标；当 density>50‰ 时触发早期干预

**Spec 3: C3-META块设计改进**
- slug: `improve-meta-block-design`
- 状态：**当前行为是设计如此**（`SKILL.md:129`）。改进方向：

- 关键文件：`skills/shenbi-chapter-drafting/SKILL.md:95-130`, `src/shenbi/gates/shared.py:120-121`

### 🟠 HIGH（5个）

**Spec 4: H1-decisions.json格式损坏**
- slug: `fix-decisions-json-corruption`
- 根因：44个decisions.json有 trailing markdown 或多JSON拼接。G2（`g2.py:85`）和G4（`decisions_validator.py:53`）的 `json.loads()` 应捕获但未捕获 — 暗示门禁未运行或被绕过

- 修复方向：增加 dispatch_helper 写前JSON验证；强化G2对"多JSON拼接"的检测；调查门禁为何未触发

**Spec 5: H2-修订系统全面失效**

- 根因：34个 revision-decisions.json — 10个JSON语法错误 + 24个零变更。`g4_decisions` 未捕获JSON错误（可能因G4路由未覆盖revision skill）

**Spec 6: H3-上下文组装持久化缺口**
- slug: `fix-context-assembly-persistence-gap`
- 根因：`context_assemble.py` 和 `context_curation.py` 确定性生成 `context/chapter-N-context.md`，但43章缺失此文件。Step 4的 `_run_context_assembly` 可能未将输出写入磁盘
- 关键文件：`src/shenbi/pipeline/chapter_loop.py:128-133`, `src/shenbi/pipeline/context_assemble.py:209-269`
- 修复方向：确保 `context/chapter-N-context.md` 在每章step 4持久化；增加 pipeline 启动时的 context 覆盖审计

**Spec 7: H4-暂存文件泄漏**
- slug: `fix-staging-residue-cleanup`
- 根因：`clear_staging()`（`checkpoint.py:59-73`）在 checkpoint approve/reject 时调用，但 `--auto` 模式（`chapter_loop.py:516-545`）直接 commit 后未显式 cleanup
- 关键文件：`src/shenbi/pipeline/checkpoint.py:20-73`, `src/shenbi/pipeline/chapter_loop.py:516-545`
- 修复方向：auto-commit 后增加 `clear_staging()` 调用；pipeline 启动时清理残留 staging

**Spec 8: H5-状态机current_step损坏**

- 修复方向：确保 `_advance` 的所有退出路径设置 `current_step`；增加状态加载时的自愈逻辑
