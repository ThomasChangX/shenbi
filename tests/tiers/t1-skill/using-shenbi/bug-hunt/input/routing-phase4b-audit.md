# 触发测试：Phase 4b 条件审计（12技能路由）

测试目标：验证 `using-shenbi` 在收到条件审计相关请求时，能正确路由到 Phase 4b 的 12 个条件审计技能。

## 测试矩阵

每个用例提供用户输入和期望路由。Agent 必须正确识别并路由。

### 用例 1: 世界规则审计

**输入:** "第18章里林轩突然用了一个9级技能，但设定里他才5级。帮我检查设定冲突。"
**期望路由:** `shenbi-review-world-rules`
**关键触发词:** 设定冲突

### 用例 2: 对话风格审计

**输入:** "我觉得最近几章角色说话越来越像了，分不清谁是谁。帮我检查对白风格一致性。"
**期望路由:** `shenbi-review-dialogue`
**关键触发词:** 对白风格、角色说话

### 用例 3: 动机审计

**输入:** "反派突然帮主角了，但前面没铺垫他为什么要这么做。帮我审查行为动机。"
**期望路由:** `shenbi-review-motivation`
**关键触发词:** 动机、行为、为什么

### 用例 4: POV 审计

**输入:** "这章用的是林轩的视角，但他怎么会知道苏婉在另一个房间里想什么？帮我检查视角问题。"
**期望路由:** `shenbi-review-pov`
**关键触发词:** 视角、知道什么

### 用例 5: 文理审计

**输入:** "这章感觉像流水账，一段接一段没什么节奏变化。帮我检查写作质感。"
**期望路由:** `shenbi-review-texture`
**关键触发词:** 流水账、节奏

### 用例 6: 爽点审计

**输入:** "第20章的高潮感觉很平，蓄了半天压但爆发不出来。帮我查爽点虚化问题。"
**期望路由:** `shenbi-review-highpoint`
**关键触发词:** 高潮、蓄压、爆发、爽点

### 用例 7: 跨章重复审计

**输入:** "我总觉得最近5章的用词和意象在重复，有什么办法检测一下？"
**期望路由:** `shenbi-review-long-span`
**关键触发词:** 跨章、重复、5章

### 用例 8: 时代审计

**输入:** "这是本明朝背景的小说，帮我检查第3章有没有出现不该有的现代词汇或器物。"
**期望路由:** `shenbi-review-era`
**关键触发词:** 时代、明朝、现代词汇

### 用例 9: 同人审计

**输入:** "这是火影忍者的同人，帮我检查第5章里鸣人的性格是不是和原作差太远了。"
**期望路由:** `shenbi-review-fanfic`
**关键触发词:** 同人、原作

### 用例 10: 外传审计

**输入:** "这是正传的外传，帮我检查有没有泄漏正传后面才出现的信息。"
**期望路由:** `shenbi-review-spinoff`
**关键触发词:** 外传、正传、泄漏

### 用例 11: 读者拉力审计

**输入:** "这几章的开头都很平淡，读者可能看两段就不想看了。帮我检查章首钩子和章尾悬念。"
**期望路由:** `shenbi-review-reader-pull`
**关键触发词:** 章首、章尾、钩子、悬念

### 用例 12: 备忘合规审计

**输入:** "第22章好像偏离了章节备忘里定的事情。帮我检查备忘合规性。"
**期望路由:** `shenbi-review-memo-compliance`
**关键触发词:** 备忘、偏离、合规

## 通过条件

- [ ] 12/12 用例正确路由到对应审计技能
- [ ] 不混淆类似技能（如 world-rules vs continuity，highpoint vs pacing）

## 失败条件

- 任何用例路由到错误的审计技能 → FAIL
- 用例 1 路由到 `shenbi-review-continuity`（世界规则不是连续性） → FAIL
- 用例 6 路由到 `shenbi-review-pacing`（爽点不是节奏） → FAIL
