# 触发测试：Phase 4c-4i 非审计技能（管理/创作扩展/短篇/导入）

测试目标：验证 `using-shenbi` 对 Phase 4c-4i 新增的非审计技能的路由准确性。

## 测试矩阵

### 用例 1: 字数调整

**输入:** "第12章才2800字，目标是4000字，帮我扩写一下。"
**期望路由:** `shenbi-length-normalizing`
**不应路由到:** `shenbi-style-polishing`（那是润色，不是调字数）

### 用例 2: 反检测

**输入:** "平台刚上了AIGC检测，这章AI味太重了，帮我做反检测改写。"
**期望路由:** `shenbi-anti-detect`
**不应路由到:** `shenbi-style-polishing`（那是润色，不是反检测）

### 用例 3: 章节模式

**输入:** "帮我看一下最近10章的章节结构是不是太单调了，检测一下章节模式。"
**期望路由:** `shenbi-chapter-pattern`

### 用例 4: 纠偏传导

**输入:** "第18章的审计结果出来了，把这些纠偏意见传导给下一章。"
**期望路由:** `shenbi-drift-guidance`

### 用例 5: 题材配置

**输入:** "帮我把疲劳词列表更新一下，再加几个玄幻常见的AI味词。顺便激活伏笔审计维度。"
**期望路由:** `shenbi-genre-config`

### 用例 6: 短篇大纲

**输入:** "我想写一个20章的短篇玄幻，帮我出个大纲。"
**期望路由:** `shenbi-short-outline`
**不应路由到:** `shenbi-story-architecture`（那是长篇框架）

### 用例 7: 短篇批量写作

**输入:** "短篇大纲做好了，帮我批量写完所有章节。"
**期望路由:** `shenbi-short-drafting`

### 用例 8: 短篇包装

**输入:** "短篇写完了，帮我生成简介、卖点、封面提示词。"
**期望路由:** `shenbi-short-packaging`

### 用例 9: 原作导入

**输入:** "我有一本已经写了50章的小说手稿，帮我导入分析一下。"
**期望路由:** `shenbi-import-analysis`

### 用例 10: 风格学习

**输入:** "我喜欢这位作者的文字风格，帮我提取他的文风指纹。"
**期望路由:** `shenbi-style-learning`

### 用例 11: 正典导入

**输入:** "我在写斗破苍穹的同人，帮我导入原作设定。"
**期望路由:** `shenbi-canon-import`

### 用例 12: 卷纲要

**输入:** "帮我规划第一卷的卷纲，每章要干什么写清楚。"
**期望路由:** `shenbi-volume-outlining`

### 用例 13: 市场趋势

**输入:** "现在起点最火的题材是什么？帮我分析一下平台趋势。"
**期望路由:** `shenbi-market-radar`

### 用例 14: 续写

**输入:** "之前写了30章停了半年，现在想继续写。帮我从断点续写。"
**期望路由:** `shenbi-sequel-writing`

## 通过条件

- [ ] 14/14 用例正确路由到对应技能
- [ ] 短篇技能（6/7/8）不路由到长篇对应技能
- [ ] 导入技能（9/10/11）互相不混淆

## 失败条件

- 用例 6 路由到 `shenbi-story-architecture` → FAIL
- 用例 1 路由到 `shenbi-style-polishing` → FAIL
- 用例 2 路由到 `shenbi-review-anti-ai` → FAIL（那是审计，不是改写）
