# 行为测试：角色一致性审计 — OOC 行为

测试目标：验证 `shenbi-review-character` 技能能在章节中检测出**主角 OOC 行为**——已建立"谨慎、谋定后动"性格的角色在本章做出与角色画像严重矛盾的孤身冒进决策。

## 测试场景

本测试提供一个章节节选，其中**刻意植入**了一个角色 OOC bug：主角林轩在角色档案中被明确描述为"谨慎、从不冒进、凡事谋定后动"，但本章他在听到求救后**未做任何信息核实、未通报同门、未等支援**，直接拔剑冲入已知有魔修埋伏的危险区域——中间无任何"动机变化"叙事支撑此行为转变。正确加载角色一致性审计技能的 agent 必须**报告这个 OOC 行为**为 `error` 级别（不是 warning），并给出具体修复建议。

## 准备

- 小说项目存在 `characters/protagonist.md`、`truth/character_matrix.md`、`truth/emotional_arcs.md`
- `protagonist.md` 记录：林轩，性格"谨慎、从不冒进"，战斗哲学"不打无准备之仗"，行为特征"凡事谋定后动"
- `truth/emotional_arcs.md` 记录：
  - 第9章末尾：林轩结束闭关，决心"谋定后动"
  - 第10章末尾：与周元合作处理外门事务，团队协作、谨慎评估
- `truth/character_matrix.md` 记录林轩的 voice_profile：内心盘算丰富，措辞审慎

## 输入（章节节选）

```markdown
# 第11章 孤身涉险

内门广场上，几个师弟正在低声议论——前几日落云谷出现了魔修活动的迹象，掌门已经下令封锁附近区域，任何弟子不得擅入。

"落云谷那边是陈师兄在负责探查，"周元走过来，压低声音，"听说是筑基后期的魔修，至少有三个人。"

林轩点了点头。他想起上个月师父的叮嘱：若遇魔修踪迹，先回报门派，切勿单独行动。他原本打算在确认安全之后才去查探。

"林师兄！林师兄！"

远处传来一声急促的呼喊。是赵小琴师妹，她的眼眶发红，衣衫上沾了血迹。

"师弟……苏师姐被困在落云谷深处了！"

林轩没再多问。他拔剑而起，身形一闪便冲向了谷口，连周元在身后喊他都没回头。

周元愣了一瞬，旋即追了上去："林师弟！落云谷有埋伏！"

但林轩的速度太快了，转眼已经消失在谷口的雾气中。

谷内，三道黑影从雾中闪出，将他团团围住。
```

## 植入的 Bug

| 位置 | 文本 | 角色档案预期 | 实际意图 |
|------|------|------------|---------|
| 第6段 | "林轩没再多问。他拔剑而起，身形一闪便冲向了谷口" | 角色档案"凡事谋定后动"，上章末尾"团队协作、谨慎评估" | 无信息核实 → 无门派通报 → 无支援等待 → 孤身冲入已知有"筑基后期魔修 × 3"的危险区域 |

**问题**：本章中林轩的行为直接违反其角色档案中"谨慎、从不冒进、凡事谋定后动"的核心特征。文中**没有任何**动机变化叙事（无信念危机、无外部压力升级、无角色成长触发事件）来支撑此行为转变——这属于典型的 OOC 写崩，而非合理的弧线推进。

## 期望的审计输出

```markdown

**结果**: 不通过

| 配角 | 降智? | 工具人? | 独立动机? |
|------|-------|---------|----------|

### 弧线
近3章林轩情感/认知状态：
- 第9章：刚结束闭关，决心"谋定后动"
- 第10章：与周元合作处理外门事务，体现"团队协作、谨慎评估"
- 第11章（本章）：突然孤身冲入危险区域，**无铺垫的弧线突变** → 弧线平坦前出现断崖式跃迁，ERROR。

### 评分: 2/10 不通过

### 建议修复
- [ERROR] 第6段林轩 OOC 意图矛盾：删除"林轩没再多问。他拔剑而起，身形一闪便冲向了谷口，连周元在身后喊他都没回头"，改为内心自我审慎 + 行动延宕，例如："林轩按住剑柄，没有立刻动作。'苏师姐被困'——是真的，还是陷阱？他压下心头的焦躁，沉声问：'赵师妹，你怎么从落云谷出来的？那三个魔修何在？'"
```

## 通过条件

- [ ] 审计报告识别出林轩孤身冲入落云谷为 OOC 行为
- [ ] 标记为 `ERROR` 级别（不是 `WARNING`）—— 铁律1: OOC = blocking error
- [ ] 章节评分 ≤ 5/10
- [ ] 给出具体修复建议（指出第6段林轩的意图与角色档案"谨慎、谋定后动"矛盾）
- [ ] BDI 评估中"意图"一致性标记为 ERROR
- [ ] 识别出本章无动机变化叙事支撑行为转变

## 失败条件

- 未检测出 OOC 行为 → FAIL
- 检测出但标记为 `WARNING` 而非 `ERROR` → FAIL（铁律1: OOC = blocking error，不可降级）
- 报告含糊未指明具体段落 → FAIL
- 未给出修复建议 → FAIL
- 用"角色可以灵活处理"或"剧情需要"等理由放过此 OOC → FAIL
