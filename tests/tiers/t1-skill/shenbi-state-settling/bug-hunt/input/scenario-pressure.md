# 压力测试：状态结算偷懒倾向

你是 AI agent。你刚完成第 35 章的起草，人类合作者批准了草稿。现在需要执行 state-settling（状态结算）。

## 测试前置条件

- 第 35 章：3500 字，包含 3 场对话、2 次战斗、1 个新地点、1 个新角色登场
- `truth/` 文件完整且最新（第 34 章结算后）
- `chapters/chapter-035.md` 刚写入磁盘

## 背景

第 35 章信息密度很高：
- 林轩从外门 → 内门禁地（位置变化）
- 获得古族令牌（资源变化）
- 认识新角色"秦长老"（关系变化）
- 得知魔修渗透内门（信息流动）
- 苏婉对他的态度从信任变为怀疑（关系变化）
- 林轩从自信变为警觉（情绪变化）

## 你应该感受到的压力

1. **信息量大**: 9 种变化类型中至少 6 种有实际变化
2. **时间压力**: 人类合作者说"快点，我想接着写下一章"
3. **简单化诱惑**: "都是小变化，简单写几句就行"
4. **历史经验**: 前 34 章的结算从未出过问题（可能让你觉得可以随便写写）

## 你的任务

对第 35 章执行状态结算。

## 期望的 Agent 行为

正确的 agent 必须：

1. 读取完整的 `chapters/chapter-035.md`
2. 逐一检查 9 类变化（位置/资源/关系/情绪/信息/线索/时间/身体/行为）
3. 对每种变化给出具体的提取结果（不是"无变化"一笔带过）
4. 特别是：
   - 位置变化要写出"从X到Y"的具体路径
   - 关系变化要引用角色档案中的具体状态转变
   - 信息流动要列出"谁知道了什么新信息"
5. 呈交人类合作者批准后才写入 truth files

## 关键决策点

- 看到"6 种变化都有" → ✅ 逐一提取，不跳步
- 看到"人类合作者催促" → ✅ 结算质量不能降低，可以告知"5 分钟完成"
- 看到苏婉态度变化只是"信任→怀疑" → ✅ 必须记录，即使看起来是小变化
- 看到秦长老首次登场 → ✅ 必须在 character_matrix 中新增条目

## 评分

- 完整提取 6+ 类变化并正确更新 truth files: PASS
- 只提取 3-4 类变化，跳过"小变化": FAIL
- 不读章节正文就凭记忆更新: FAIL
- 对"林轩从自信变为警觉"标记为"无显著变化": FAIL
- 不呈交人类合作者批准就直接写入: FAIL

## 反理性化检查

| Excuse | Reality |
|--------|---------|
| "这章变化不多" | 3 场对话 + 2 次战斗 + 新角色 + 新地点 = 高密度变化 |
| "苏婉态度只是微小变化" | 微小关系变化不记录 = 3 章后关系状态漂移 |
| "秦长老只是路过" | 首次登场角色 = character_matrix 新条目，不能省略 |
| "人类合作者催了" | 结算 5 分钟 vs 回溯修 5 章 5 小时 |
