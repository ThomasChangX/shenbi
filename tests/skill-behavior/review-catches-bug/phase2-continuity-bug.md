# 行为测试：连续性审计 — 时间线矛盾

测试目标：验证 `shenbi-review-continuity` 技能能在章节中检测出**时间线矛盾**（一天内太阳位置自相矛盾）。

## 测试场景

本测试提供一个章节节选，其中**刻意植入**了一个时间线 bug：角色在正午到达某地，但仅两段之后文本却描述"太阳正在西沉"。正确加载连续性审计技能的 agent 必须**报告这个矛盾**为 `error` 级别（不是 warning）。

## 准备

- 小说项目存在 `truth/current_state.md` 和 `truth/chapter_summaries.md`
- `current_state.md` 记录：第5章末尾，主角林轩在内门演武场；时间标签 = "正午"
- `chapter_summaries.md` 近3章无时间线异常

## 输入（章节节选）

```markdown
# 第6章 演武风云

正午的阳光直直地砸在青石板上，林轩踏入了内门演武场。四周的师兄们正三三两两地席地而坐，等候着下午的考核开始。

他在场边站定，深吸一口气。今日的对手，是那个半月前在内门大会上出尽风头的周元。

比试未起，风已先行。林轩余光扫过场地另一侧，看见了那个熟悉的身影——周元正抱臂而立，嘴角挂着一丝若有若无的笑意。

"来了？"周元淡淡开口。

林轩没有答话，只是把手中长刀横在身前。

这时候太阳已经西沉，暮色从演武场四周的林间漫了上来。周元的笑意在昏黄的光线里变得有些模糊。

"开始吧。"长老在阴影里抬手。
```

## 植入的 Bug

| 位置 | 文本 | 推断时间 |
|------|------|---------|
| 第1段 | "正午的阳光" | 12:00 |
| 第5段 | "太阳已经西沉" / "暮色从演武场四周的林间漫了上来" | 17:00-18:00 |

**问题**：同章内，从"正午"到"日落"之间没有任何时间过渡叙事（无"过了几个时辰"、无"日头渐斜"等衔接），且中间没有标注"次日"或"数日后"等明确时间跳跃。事件链要求这段时间内完成"入场 → 等待 → 对峙 → 长老宣布开始"——而这几件事显然不可能在正午到日落的同一场内连续发生（中间有"下午的考核"作为暗示时间节点）。

## 期望的审计输出

```markdown
## 连续性审计报告

**章节**: 第6章
**结果**: 不通过

### 时间线
| 标记 | 位置 | 推断时间 | 上一章末尾时间 | 状态 |
|------|------|---------|-------------|------|
| "正午的阳光" | 第1段 | 正午 | 正午（第5章末尾） | OK |
| "太阳已经西沉" | 第5段 | 17:00-18:00 | — | MISMATCH |
| "暮色从演武场四周的林间漫了上来" | 第5段 | 17:00-18:00 | — | MISMATCH |

### 地点
演武场（无跳跃，OK）

### 事件时序
1. 林轩入场 — 正午
2. 等候考核 — 正午
3. 与周元对峙 — 推断在下午
4. 长老宣布开始 — 推断在下午
5. "太阳已经西沉" — 17:00-18:00

事件 1-2 紧接事件 3-5 缺乏时间过渡叙事。同章内从"正午"到"日落"无明确时间标记。

### 物理空间
OK

### 评分: 3/10 不通过

### 建议修复
- [ERROR] 第5段时间矛盾：在"长老在阴影里抬手"前补一段过渡叙事，明确时间从下午推进到傍晚，或在第5段前增加"数个时辰过去"等时间标记
```

## 通过条件

- [ ] 审计报告识别出"正午"与"太阳西沉"之间的时间矛盾
- [ ] 标记为 `ERROR` 级别（不是 `WARNING`）
- [ ] 章节评分 ≤ 5/10
- [ ] 给出具体修复建议（指出第5段需补充时间过渡）

## 失败条件

- 未检测出时间矛盾 → FAIL
- 检测出但标记为 `WARNING` 而非 `ERROR` → FAIL（这是硬错误，不是建议）
- 报告含糊未指明具体段落 → FAIL
