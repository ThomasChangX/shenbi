# 行为测试：读者拉力审计 — 章首章尾无钩子

测试目标：验证 `shenbi-review-reader-pull` 能检测出章首缺乏吸引力和章尾缺乏悬念——读者可能在开头就离开，或在结尾没有翻到下一章的冲动。

## 输入（章节）

```markdown
# 第16章

上一章的战斗结束了，林轩回到了宿舍。他坐在床边，回忆着今天的经历。

他想着想着就睡着了。第二天一早，他像往常一样起床洗漱。

他去了食堂吃了早饭。粥和馒头，味道一般。

吃完早饭他去练功房。路上遇到几个同门，互相打了个招呼。

练功结束后他去藏书阁看书。看了半个时辰觉得累了就回宿舍了。

他躺在床上想，今天过得挺平淡的。也许明天会好一点。

他把灯吹灭了，闭上了眼睛。
```

## 植入的 Bug

| 维度 | 问题 | 严重度 |
|------|------|--------|
| 章首钩子 | 第一句"上一章的战斗结束了，林轩回到了宿舍"——无悬念、无冲突、无新信息、无好奇心 | error |
| 章尾悬念 | 最后一句"也许明天会好一点"——无未解问题、无危险预告、无期待建立 | error |
| 期待管理 | 全章无任何读者期待（无承诺、无延迟、无兑现） | error |

## 通过条件

- [ ] 检测出章首无钩子
- [ ] 检测出章尾无悬念
- [ ] 两项均标记为 `error`
- [ ] 建议具体的章首/章尾改进方案

## 失败条件

- 未检测出章首问题 → FAIL
- 未检测出章尾问题 → FAIL
- 认为"过渡章可以这样" → FAIL（铁律: 每章都应有章首钩子和章尾悬念）
