# 行为测试：伏笔审计 — 备忘兑现缺失 + 培育间隔过期

测试目标：验证 `shenbi-review-foreshadowing` 技能能在章节中检测出两类伏笔债务：(1) 备忘承诺兑现的伏笔在正文中缺失（memo mismatch），(2) 伏笔超过 `cultivation_interval` 未强化（cultivation overdue）。

## 测试场景

本测试提供第 100 章的场景，**刻意植入**两个伏笔 bug：

- **Bug 1（备忘兑现缺失）**：`plans/chapter-100-plan.md` 的备忘 hook 账中明确标注 `resolve hook-001`（"青冥剑线索"，第 50 章种植），但第 100 章正文**完全没有兑现**该伏笔。备忘承诺与正文不一致。
- **Bug 2（培育间隔过期）**：`truth/pending_hooks.md` 记录 `hook-002`（"师父身世"）的 `last_reinforced` 是第 85 章，`cultivation_interval` 为 10，到第 100 章已 15 章未强化，**超过阈值**。

正确加载伏笔审计技能的 agent 必须**同时报告两个 bug**：备忘兑现缺失标记为 `error`（按铁律1：过期伏笔 = error + 铁律3：备忘必须与正文一致），培育间隔过期标记为 `warning`（按铁律2：培育超期需警告），并给出具体修复建议。

## 准备

- 小说项目存在 `truth/pending_hooks.md` 和 `plans/chapter-100-plan.md`
- `truth/pending_hooks.md` 关键记录：
  - `hook-001`："青冥剑线索"，状态 RELEVANT，种植章 = 50，`max_distance` = 30，`last_reinforced` = 70
  - `hook-002`："师父身世"，状态 PLANTED，种植章 = 60，`cultivation_interval` = 10，`last_reinforced` = 85
- `plans/chapter-100-plan.md` 的备忘 hook 账：
  - `open hook-003` "黑雾真相"（新伏笔种植）
  - `resolve hook-001` "青冥剑线索"（承诺本章兑现）

## 输入（章节 100 节选）

```markdown
# 第100章 黑雾初现

落云谷深处，黑雾从地底裂缝中涌出，带着一种腐蚀性的气味。

林轩用袖口掩住口鼻，眯眼观察这股黑雾的流动方向。三天前他在北云山矿脉发现过类似的痕迹——当时他在废矿第七层记录过一条旁注：**黑雾来源或与古传送阵有关**。这件事他一直记在心里。

"林师兄，这雾有毒吗？"周元在身后压低声音。

"不致命，但长时间接触会侵蚀灵力。"林轩头也不回地答道，"而且它的流向很有规律——像是有东西在底下引导它。"

周元没有再说话，只把腰间的解毒丹分了一半递给林轩。

两人继续朝黑雾深处推进。林轩隐约感到，黑雾背后可能藏着一个比落云谷本身更古老的秘密——也许和上古宗门的覆灭有关，也许和青冥剑失踪的真相有关。

他停下脚步，从怀中取出一块留影玉简，把黑雾的形态仔细录了下来。

"先把这一段记录下来，"他对自己说，"回去之后再分析。"

远处，黑雾深处传来一声低沉的轰鸣。
```

## 植入的 Bug

| 维度 | 备忘承诺 | 实际正文 | 状态 |
|------|---------|---------|------|
| hook-001 兑现 | `resolve hook-001`（青冥剑线索） | 仅在第 10 段提及"青冥剑失踪的真相有关"作为联想，**未实际兑现**伏笔，无解释线索含义、无揭示真相 | MISMATCH |
| hook-002 培育 | 无强化要求（备忘录未列入本章动作） | 第 100 章（= 85 + 15），超过 `cultivation_interval: 10` | OVERDUE |
| hook-003 种植 | `open hook-003`（黑雾真相） | 第 1 段描述黑雾现象，作为新伏笔植入 ✓ | MATCH |

**问题 1**：备忘承诺 `resolve hook-001` 但正文未兑现。备忘账本与正文不一致（铁律3），且 hook-001 种植于第 50 章、`max_distance` = 30，到第 100 章（差 50 章）已经远超 `max_distance`，属于过期伏笔（铁律1：error）。

**问题 2**：hook-002 自第 85 章起未强化，第 100 章时已间隔 15 章，超过 `cultivation_interval: 10`，属于培育超期。

## 通过条件

- [ ] 审计报告识别出备忘中 `resolve hook-001` 在正文中未兑现
- [ ] 备忘兑现缺失标记为 `ERROR` 级别（按铁律1：过期伏笔 = error + 铁律3：备忘必须与正文一致）
- [ ] 审计报告识别出 hook-001 超过 `max_distance: 30`（种植 50，当前 100，间隔 50）
- [ ] max_distance 违反标记为 `ERROR` 级别（按铁律1：过期伏笔 = error）
- [ ] 审计报告识别出 hook-002 超过 `cultivation_interval: 10`（上次强化 85，当前 100，间隔 15）
- [ ] 培育间隔过期标记为 `WARNING` 级别（按铁律2：超期需警告）
- [ ] 章节评分 ≤ 5/10（不通过）
- [ ] 给出具体修复建议，分别针对 hook-001 备忘缺失、hook-001 距离过期、hook-002 培育超期
- [ ] 建议修复条目以 `[ERROR]` 或 `[WARNING]` 前缀标注严重度

## 失败条件

- 未检测出备忘 hook-001 兑现缺失 → FAIL
- 未检测出 hook-001 超过 `max_distance` → FAIL
- 未检测出 hook-002 培育超期 → FAIL
- 备忘兑现缺失标记为 `WARNING` 而非 `ERROR` → FAIL（备忘与正文脱节 = 伏笔账本失效，是硬错误）
- 培育超期标记为 `ERROR` 而非 `WARNING` → FAIL（培育超期是 warning，不是阻塞级 error）
- max_distance 违反未标注 `ERROR` → FAIL（铁律1：过期伏笔 = error）
- 报告含糊未指明具体 hook ID（hook-001 / hook-002）→ FAIL
- 未给出修复建议 → FAIL
- 用"读者记不住之前的伏笔"或"备忘差一点没关系"等理由放过此问题 → FAIL
