# 行为测试：读者牵引力审计 — 章尾无悬念

测试目标：验证 `shenbi-review-reader-pull` 技能能在章节中检测出**章尾悬念彻底缺失**的 bug——章节以"平静入睡"收束，没有任何钩子、悬念、转折或期待管理。

## 测试场景

本测试提供一个章节节选，其中**刻意植入**了一个读者牵引力 bug：本章是承接上一章高潮（苏婉重伤）后的过渡章，但全章既无章首钩子（开头直接描写日常），又无章尾悬念（结尾主角"一夜无梦"入睡收束），全章读起来像断电的流水账。正确加载读者牵引力审计的 agent 必须**报告这个牵引力真空**为 `error` 级别。

## 准备

- 小说项目存在 `plans/chapter-008-plan.md`、`truth/pending_hooks.md`
- `chapter-008-plan.md` 第2段（读者期待）记录："本章需要为下一章的师门问责铺垫期待点"
- `truth/pending_hooks.md` 记录至少 2 个 ACTIVE 钩子（"师门问责" + "魔修余党动向"）等待本章推进

## 输入（章节节选）

```markdown
# 第8章 平静的一天

第二天一早，林轩醒过来的时候，窗外已经大亮。

他起身洗漱，整理了一下衣冠，然后走出厢房。

外门的演武场上，几个师弟正在练剑。林轩看了一会儿，觉得招式都还挺扎实，就点了点头，转身往藏书阁走去。

藏书阁里人不多。林轩挑了一本《灵草图鉴》，坐在角落里翻了起来。书上说九叶青这种灵草只生长在海拔三千丈以上的阴面山坡，他记下了这个知识点。

午后回到厢房，林轩又睡了一个长长的午觉。

傍晚时分，他去饭堂吃了晚饭，今天的菜是清炒灵笋和一碗灵米饭，味道还不错。

吃完饭，林轩回到厢房，掌灯看了一会儿书，就熄灯睡下了。

一夜无梦。
```

## 植入的 Bug

| 位置 | 维度 | 期望 | 实际 | 状态 |
|------|------|------|------|------|
| 开头第1段 | 章首钩子 | 应有钩子（场景冲击 / 悬念 / 反差） | "第二天一早，林轩醒过来"（典型流水账开头） | **缺失** |
| 中段 | 期待管理 | 应推进"师门问责"或"魔修余党"钩子 | 全章无任何相关伏笔触发 | **缺失** |
| 章尾最后一段 | 章尾悬念 | 应有未解之钩、转折、期待 | "一夜无梦"（完全闭合，无任何期待） | **缺失** |
| 整体 | 主动词密度 | ≥ 3 处动作/事件推进 | 0 处（起床、看弟子上课、看书、睡觉、吃饭） | **缺失** |

**问题**：
1. 上一章是苏婉重伤、林轩涉险的高潮，本章完全没有承接——像换了本书。
2. 备忘第2段明确写"为下一章师门问责铺垫期待点"，但全章无任何关于"问责即将发生"的暗示。
3. 章尾"一夜无梦"是**反钩子写法**——读者没有理由点开下一章。
4. 全章是典型的"日历式叙述"（起床→看书→吃饭→睡觉），无任何戏剧动作。

```markdown

| 章尾悬念 | 3 | 0 | 缺失 |
| **合计** | **10** | **0** | 不通过 |

### 详细分析

**章首钩子（0/3）**
- 实际开头："第二天一早，林轩醒过来的时候，窗外已经大亮。"
- 问题：典型的"日历开头"——告诉读者"这一天开始了"，无场景冲击力。
- 建议：以一句对话或异常事件开头。例如："林轩推开藏书阁的门，却看见陈长老正坐在他的位子上翻看《灵草图鉴》。"

**期待管理（0/4）**
- 备忘承诺：为"师门问责"铺垫期待
- 实际：全章无任何关于问责的暗示
- 建议：至少在某处（弟子的窃窃私语、长老派人的脚步声、一封突然送到的信笺）植入"问责即将发生"的信号

**章尾悬念（0/3）**
- 实际结尾："一夜无梦。"
- 问题：完全闭合 + 反钩子表达（"无梦"= 没事发生）
- 建议：章尾必须留一个未解之钩。例如："林轩睡着的时候，不知道药童阿七在医庐里发现了一行血字——那是魔修的暗记。"

### 评分: 0/10 不通过

### 建议修复
- [ERROR] 重写章首：从"林轩推开藏书阁门"开始，引入一个异常细节（陈长老在场、书被换过、信件突然出现等）
- [ERROR] 重写中段：至少植入 1 处"师门问责即将发生"的信号
- [ERROR] 重写章尾：删去"一夜无梦"，改为一个未解之钩（建议保留"睡去"动作，但保留悬念元素）
- [ERROR] 整体：本章必须有至少 1 个戏剧动作（不能全是日常流水）
```

## 失败条件

- 未检测出章尾无悬念 → FAIL
- 检测出但说"过渡章可以平淡一些" → FAIL（备忘已签字画押要求铺垫）
- 报告说"本章基调平稳，读者可以接受" → FAIL
- 没有引用备忘承诺 → FAIL
- 评分 ≥ 5/10 → FAIL
