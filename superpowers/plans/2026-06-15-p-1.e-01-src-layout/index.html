    P-1.E Plan 1: src/shenbi/ Layout Migration (PR-18 + PR-47)

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

    Goal: Migrate Shenbi framework runtime code from tests/ to src/shenbi/ (PR-18), then remove force-include block from wheel build (PR-47). This is the blocking prerequisite for all other P-1.E plans.

    Architecture: Atomic src-layout migration. Files move with git mv to preserve history. Hyphenated filenames (phase-runner.py, summarize-round.py, update-progress.py) are renamed to valid Python module names (underscores). tests/validate-gate.py (4318 lines) and tests/dispatch-subagent.sh (203 lines) stay in place — they're handled by PR-19 and PR-20 in Plan 2. The forwarder stubs in tests/gates/cli.py and tests/dispatcher/cli.py move WITH path fixups (their __file__-relative paths change after relocation). Six [project.scripts] entry points declared; the seventh (shenbi-generate-plugins) waits for PR-45. No behavior changes — modules retain identical runtime behavior, but __file__-relative path computations in 4 runtime modules + 2 forwarder stubs are updated to resolve to the repo root from their new src/shenbi/ location.

    Tech Stack: Python 3.11+, uv 0.10+, hatchling build backend, pyproject.toml (PEP 621).

    Spec reference: docs/superpowers/specs/2026-06-15-p-1.e-foundation-completion/02-structural-layout.md (Cluster 02, PR-18 + PR-47)

    Depends on: Nothing (this is the first P-1.E plan).

    Blocks: All other P-1.E plans (PR-19 through PR-46 assume src/shenbi/ exists).

    Estimated effort: L (3-4 hours, 20 tasks)
