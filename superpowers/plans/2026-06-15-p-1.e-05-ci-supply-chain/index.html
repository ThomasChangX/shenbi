    P-1.E Plan 5: CI / Supply Chain Hardening (PR-35 ~ PR-39)

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans.

    Goal: Harden CI: add uv lock --check, enable cache, add Python 3.13, add CodeQL, fix renovate policy, automate pre-commit autoupdate, create release workflow with SBOM attach.

    Architecture: Five independent PRs touching only .github/workflows/ + renovate.json + .pre-commit-config.yaml. No source code changes.

    Tech Stack: GitHub Actions, uv 0.10+, CodeQL, Renovate, softprops/action-gh-release.

    Spec reference: docs/superpowers/specs/2026-06-15-p-1.e-foundation-completion/05-ci-supply-chain.md

    Depends on: Plan 1 PR-18 (src layout) + Plan 2 PR-23 (mutation comparator + baseline) + Plan 3 PR-25 (remove mypy overrides).

    Blocks: None (CI work doesn't block framework functionality).

    Estimated effort: M (3-5 days).

    PR-35: CI Hardening (uv lock check + cache + Python 3.13 + mutation)

    Task 1: Update .github/workflows/ci.yml

    Files: - Modify: .github/workflows/ci.yml

    • [ ] Step 1: Add uv lock --check + cache + Python 3.13 + mutation job

    Preserve the name:, on:, and concurrency: blocks above jobs: (the current ci.yml has name: CI, push/PR triggers, and a concurrency group with cancel-in-progress: true). Replace only from jobs: to EOF with:

    jobs:
  quality:
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        python-version: ["3.11", "3.12", "3.13"]
    # 3.13 allow-failure during rollout (spec Risk table).
    # Windows allow-failure until @pytest.mark.skip_on_windows marker is
    # registered (spec OQ-1; deferred to a follow-up source-code PR).
    continue-on-error: ${{ matrix.python-version == '3.13' || matrix.os == 'windows-latest' }}
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true
          cache-dependency-glob: "uv.lock"
      - run: uv python install ${{ matrix.python-version }}
      - name: Verify uv.lock is up to date
        run: uv lock --check
      - run: uv sync --frozen --extra dev
      - run: uv run ruff check .
      - run: uv run ruff format --check .
      - run: uv run mypy src/shenbi/
      - run: uv run basedpyright
      - name: Run tests (parallel, excluding last)
        run: uv run pytest -n auto -m "not last" --hypothesis-profile=ci
      - name: Run coverage threshold test (serial, last only)
        run: uv run pytest -p no:xdist -m "last" --no-cov --hypothesis-profile=ci

  dependency-review:
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4
      - name: Dependency review (license + vulnerability)
        uses: actions/dependency-review-action@v4

  action-validation:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true
      - run: uv sync --frozen --extra dev
      - name: Validate GitHub Actions YAML
        run: uv run yamllint --strict .github/workflows/

  mutation-testing:
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true
      - run: uv sync --frozen --extra dev
      - name: Run mutation testing
        run: |
          # mutmut 3.x reads source_paths from [tool.mutmut] config;
          # per-file CLI scoping is not supported (no --paths-to-mutate
          # or --use-cache flags in 3.x). Changed-file detection only
          # decides whether to run mutmut at all.
          BASE="origin/${{ github.event.pull_request.base.ref }}"
          CHANGED=$(git diff --name-only "${BASE}...HEAD" \
            -- 'src/shenbi/')
          if [ -z "$CHANGED" ]; then
            echo "No Python changes; skipping mutation testing"
            exit 0
          fi
          uv run mutmut run
      - name: Compare to baseline (blocks on regression > 5%)
        run: |
          # compare_mutation_score.py runs 'mutmut results' internally
          # and parses output; it only accepts --baseline and --threshold
          # (Plan 2 PR-23). No --current flag exists.
          uv run python tools/compare_mutation_score.py \
            --baseline tests/baselines/mutation-score.txt

    PR-39: Release Workflow + SBOM Attach

    Task 5: Create .github/workflows/release.yml

    Files: - Create: .github/workflows/release.yml

    • [ ] Step 1: Write release workflow
    name: Release
on:
  push:
    tags: ['v*']

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true
      - run: uv sync --frozen --extra dev
      - name: Generate SBOM (CycloneDX)
        run: uv run cyclonedx-py environment -o sbom.cdx.json
      - name: Build sdist and wheel
        run: uv build
      - name: Generate changelog for tag
        id: changelog
        run: |
          prev_tag=$(git describe --tags --abbrev=0 HEAD^ 2>/dev/null || echo "")
          if [ -n "$prev_tag" ]; then
            notes=$(git log --format='- %s (%h)' ${prev_tag}..HEAD)
          else
            # First release: cap to recent 100 commits, not full history
            notes=$(git log --format='- %s (%h)' -n 100)
          fi
          echo "notes<<EOF" >> $GITHUB_OUTPUT
          echo "$notes" >> $GITHUB_OUTPUT
          echo "EOF" >> $GITHUB_OUTPUT
      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          body: ${{ steps.changelog.outputs.notes }}
          files: |
            dist/*.whl
            dist/*.tar.gz
            sbom.cdx.json
