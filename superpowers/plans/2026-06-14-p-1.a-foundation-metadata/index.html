    P-1.A Foundation Metadata Implementation Plan

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

    Goal: Establish modern Python project metadata baseline (pyproject.toml + uv + ruff + mypy + basedpyright) so all subsequent P-1 PRs build on standardized tooling.

    Architecture: Single-commit-per-PR strangler fig pattern. P-1.A produces zero behavior change to existing code; it only adds config files and runs auto-fixers on existing source. Legacy modules (validate-gate.py, scoring.py, etc.) get temporary ignore_errors overrides removed in PR-12 (P-1.D).

    Tech Stack: Python ≥3.11, uv (package manager), ruff (lint+format), mypy --strict (type check), basedpyright (pure-Python pyright fork, type check)

    Spec: docs/superpowers/specs/2026-06-14-p-1-foundation-hygiene-design.md
