import sys
from typing import Any


def emit_json(data: Any) -> None:

- [ ] **Step 3: Create `tests/unit/test_test_density.py`**

Note: this version counts `ast.AsyncFunctionDef` in addition to `ast.FunctionDef`. The spec's reference implementation counts only `ast.FunctionDef`, but this project uses `asyncio_mode = "auto"` so async test functions are valid. This is a deliberate, documented improvement over the spec.

```python
"""Enforce test density floor per README Threshold Justification.

Metric: test_function_count / framework_loc
Target: >= 0.10 (1 test function per 10 LOC of framework code).
"""

import ast
from pathlib import Path

REPO_ROOT = Path(__file__).resolve().parents[2]
FRAMEWORK_DIR = REPO_ROOT / "src" / "shenbi"
TEST_DIRS = [REPO_ROOT / "tests" / sub for sub in ("unit", "integration", "property", "benchmark")]


def count_framework_loc() -> int:
    """Count non-blank, non-comment Python LOC in src/shenbi/."""
    total = 0
    for py_file in FRAMEWORK_DIR.rglob("*.py"):
        for line in py_file.read_text(encoding="utf-8").splitlines():
            stripped = line.strip()
            if stripped and not stripped.startswith("#"):
                total += 1
    return total


def count_test_functions() -> int:
    """Count `def test_*` (and `async def test_*`) functions across all test directories."""
    total = 0
    for test_dir in TEST_DIRS:
        if not test_dir.exists():
            continue
        for py_file in test_dir.rglob("*.py"):
            try:
                tree = ast.parse(py_file.read_text(encoding="utf-8"))
            except SyntaxError:
                continue
            for node in ast.walk(tree):
                if isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef)) and node.name.startswith("test_"):
                    total += 1
    return total


def test_density_meets_minimum():
    framework_loc = count_framework_loc()
    test_count = count_test_functions()
    density = test_count / framework_loc if framework_loc else 0
    assert density >= 0.10, (
        f"Test density {density:.4f} below 0.10 floor "
        f"({test_count} tests / {framework_loc} framework LOC). "
        f"Add ~{int(0.10 * framework_loc - test_count)} more tests."
    )
```

- [ ] **Step 6: Merge ordering — must land AFTER Plan 4 (Cluster 04)**

**Critical sequencing rule**: PR-26 must merge only after Plan 4 PR-28 ~ PR-32 deliver ~600 tests. If PR-26 merges before Plan 4, CI fails on `main` until Plan 4 catches up — a red `main` blocks all other PRs.

Two ways to enforce this:

1. **Preferred — land PR-26 last in P-1.E sequence**: hold the PR open until Plan 4 merges, then rebase + merge. PR-26's diff is tiny (one `fail_under` change + one new test file), so rebase cost is near zero.

2. **Alternative — temporary `xfail`**: mark `test_density_meets_minimum()` with `@pytest.mark.xfail(strict=True, reason="Plan 4 PR-28~32 will deliver remaining tests")`. This keeps the *density test* green. **Caveat**: `xfail` does NOT cover the `fail_under = 90` enforcement. pytest-cov exits non-zero at session teardown and that exit code cannot be `xfail`-ed. So the xfail option alone leaves `main` red on the coverage gate. To stay fully green you must ALSO temporarily lower `fail_under` (or pass `--no-cov-fail-under`) and revert both in the same Plan 4 PR. **Remove the xfail in the same Plan 4 PR that delivers the 600th test** (otherwise strict xfail flips to XPASS = failure). Given this caveat, option 1 (land last) is the only zero-config safe path.

If neither option is followed and `main` goes red, the fix is to revert PR-26 entirely until Plan 4 catches up — do not lower the threshold as a workaround.
