# 同人模式严格度对照

此参考由 `shenbi-review-fanfic` 技能使用。说明 4 种同人模式（canon/au/ooc/cp）的允许范围、严格度参数、判定规则。

## 1. 4 种模式概览

| 模式 | 严格度 | 主要目的 | 允许的偏差 |
|------|--------|---------|----------|
| canon | 最严 | 续写/补完原作 | 几乎不允许（需显式声明 OOC 章节） |
| au | 中 | 在 AU 设定中演绎角色 | 设定内可自由发挥；设定外需声明 |
| ooc | 宽松 | 探索角色的另一面 | 故意偏离，但需在文前/章节标注 |
| cp | 中（关系维度严） | 关系演绎 / 配对 | 配对可新创，但人物与世界观仍受 canon 约束 |

## 2. canon 模式（最严）

### 2.1 适用情境

- 续写原作未完成的故事线
- 补完原作中未展开的支线
- 探索原作已有的可能性

### 2.2 严格度参数

| 维度 | 严格度 | 偏差容忍 |
|------|--------|---------|
| 角色性格 | 严 | 几乎无 |
| 能力体系 | 严 | 几乎无 |
| 世界规则 | 严 | 几乎无 |
| 关系动态 | 严 | 需沿原作发展曲线 |
| 事件一致性 | 严 | 必须沿用 |
| 标志性特征 | 严 | 全部保留 |

### 2.3 判定规则

- 任何与原作偏差（人物性格/能力/关系/事件）= error（除非作者在 `novel.json` 中声明"本章为 OOC 偏离"）
- 偏差声明 = 显式标注（章节开头 / 文前说明 / 注释）
- 无声明的偏差 = 读者信任崩塌

## 3. au 模式（中等）

### 3.1 适用情境

- 平行世界设定（如"如果 X 当时选了 Y"）
- 职业/性别/时代替换
- 背景重构

### 3.2 严格度参数

| 维度 | 严格度 | 偏差容忍 |
|------|--------|---------|
| 角色性格 | 中 | AU 设定内允许调整，但需"同内核" |
| 能力体系 | 中 | AU 内可重新定义，但需有逻辑 |
| 世界规则 | 中 | AU 内可改，但需在 `novel.json` 声明 |
| 关系动态 | 中 | 沿 AU 设定逻辑 |
| 事件一致性 | 松 | AU 即允许偏差 |
| 标志性特征 | 中 | 全部保留可作"识别锚" |

### 3.3 判定规则

- AU 设定必须在 `novel.json` 中显式声明
- 设定内偏差 = pass
- 设定外偏差 = error
- 同内核判定：核心性格特质（如"骄傲""执着"）保留 = pass；完全反向 = error

## 4. ooc 模式（宽松）

### 4.1 适用情境

- 探索角色的非典型状态（黑化/沙雕/现代 AU）
- 反差萌 / 喜剧向
- 实验性创作

### 4.2 严格度参数

| 维度 | 严格度 | 偏差容忍 |
|------|--------|---------|
| 角色性格 | 宽松 | 故意偏离，但需标注 |
| 能力体系 | 宽松 | 可改 |
| 世界规则 | 宽松 | 可改 |
| 关系动态 | 宽松 | 可改 |
| 事件一致性 | 宽松 | 几乎无约束 |
| 标志性特征 | 中 | 仍宜保留部分 |

### 4.3 判定规则

- OOC 必须在章节开头或文前显式标注（如"OOC 警告：本章为平行娱乐向"）
- 标注后的偏离 = pass
- 无标注的偏离 = error（视为作者误操作）

## 5. cp 模式（关系维度严）

### 5.1 适用情境

- 配对文 / 恋爱演绎
- 原作关系外的 CP 拉郎
- 原作已有 CP 的展开

### 5.2 严格度参数

| 维度 | 严格度 | 偏差容忍 |
|------|--------|---------|
| 角色性格 | 中-严 | 需"可推导该 CP" |
| 能力体系 | 严 | 通常沿原作 |
| 世界规则 | 严 | 通常沿原作 |
| 关系动态 | 严 | CP 演进需合理 |
| 事件一致性 | 中 | 事件作背景，关系演绎为主 |
| 标志性特征 | 中-严 | 保留 |

### 5.3 判定规则

- CP 关系建立需有铺垫（前章互动 / 心理刻画）
- 拉郎 CP 需明确"为何这两人"——背景、关系演化、性格契合
- CP 互动不能扭曲原作其他关系（除非 AU 模式）
- 单章内关系突变 = warning（需章节内有过程）

## 6. 模式选择的实操建议

### 6.1 不确定时

- 默认 canon 模式
- 在 `novel.json` 中显式声明所选模式
- 一部作品可多章节使用不同模式，但每章需标注

### 6.2 模式切换

- 章节内模式切换需要在章节开头标注
- 突然切换 = warning（除非在 `novel.json` 预设章节级模式）

### 6.3 多模式混合

- 高级用法：例如"主 CP 章节 + 支线 AU 章节"
- 需要 `novel.json` 详细配置每章模式
- 审计时按章节所属模式严格度检查

## 7. 关键判定流程

```
1. 读取 novel.json.fanfic.mode
2. 若未声明 → 视为 canon（最严默认）
3. 加载对应模式严格度参数
4. 逐项检查（角色/能力/规则/关系/事件/特征）
5. 偏差有显式声明？→ pass
6. 偏差无显式声明？→ 按模式严格度判定严重度
```

## 8. 与其他审计协作

- `shenbi-review-character`：本审计的"角色性格/能力"与角色一致性审计部分重叠
  - 区别：本审计看"与原作对齐"，character 审计看"与作品内档案对齐"
  - 优先级：原作对齐 > 档案对齐（原作是更基础的契约）
- `shenbi-review-world-rules`：本审计的"世界规则"与世界观审计部分重叠
  - 区别：本审计看"与原作世界对齐"，world-rules 审计看"作品内自洽"
  - 优先级：原作对齐 > 作品内自洽
