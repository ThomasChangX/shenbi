# 敏感词检测参考

此参考由 `shenbi-review-sensitivity` 技能使用。提供常见平台禁忌类别、检测方法、判定阈值。

> **重要：** 本文件只描述**检测方法与类别边界**，不提供完整词表。完整词表会随平台政策更新而失效，且会变成规避工具。审计时应查阅目标平台的最新官方审核规范。

## 1. 中国网文平台禁忌类别

### 1.1 政治敏感

**类别边界**：涉及现实政治人物、政党、政权、历史事件、民族宗教、领土主权、领导人相关绰号或负面表述。

**检测方法**：
- 维护一份"高风险实体名"参考（领导人名、敏感事件名、敏感地名），仅作审计时参照
- 上下文判断：单独出现可能无害，结合特定语境则触发（如与政权、制度、合法性相关）
- 同义/变形检测：谐音、拼音、缩写、emoji 替代

**判定原则**：宁可误报不可漏报。平台对政治类违规零容忍，误报可人工复核，漏报直接下架。

### 1.2 色情描写

**类别边界**：详细性器官描写、性行为过程描写、以性吸引为唯一目的的裸露描写。

**检测方法**：
- 关键动作词密度（仅作辅助信号）
- 场景结构判断：是否构成独立色情场景（影响判定严重度）
- 主题边界：与作品主线的相关性（主线擦边 vs 独立色情桥段）

**判定原则**：依赖场景与上下文。同样的字面描述在以情节推进为目的时与独立色情桥段中处理不同。

### 1.3 极端暴力

**类别边界**：对未成年人施加的暴力、酷刑细节描写、以娱乐化方式呈现的极端血腥、真实手法还原的犯罪过程。

**检测方法**：
- 受害者身份标记（年龄、身份关键词）
- 暴力描写详细度（是否进入实施细节 vs 结果陈述）
- 场景目的（情节推进 vs 渲染血腥）

**判定原则**：以"对未受保护的受害者施加详细暴力"为绝对红线。战斗类、玄幻类暴力在主线推进语境下不触发。

## 2. 英文平台参考

### 2.1 Amazon KDP

**主要类别**：
- Adult content（明确色情）必须标注为 Adult，否则下架
- Hate speech / extremist content 零容忍
- Graphic violence 需有上下文正当性

**合规性**：不提供具体词表，依赖 Amazon 官方 Content Guidelines。检测方法参照 KDP 政策。

### 2.2 Royal Road

**主要类别**：
- Harems 与过度性化内容受严格审查
- 极端暴力（尤其针对儿童）零容忍
- 仇恨内容零容忍

**合规性**：参考 Royal Road 官方 Terms of Service。

### 2.3 Wattpad

**主要类别**：
- Mature content 必须使用年龄分级标签
- 未分级成人内容下架
- 自残/自杀描写有特殊规则

**合规性**：参考 Wattpad Content Guidelines。

### 2.4 通用英文平台

无平台指定时，应用最严标准（Amazon KDP）。`novel.json` 缺省 `targetPlatform` 时按 KDP 规则处理。

## 3. 本书禁忌词检测

**来源**：`genre-config.json` 的 `prohibitions` 数组。

**检测方法**：
- 精确字符串匹配
- 大小写不敏感（英文）
- 简体/繁体等价匹配
- 包含空白变体（"  全场震惊  " 等）

**判定阈值**：每章必须为 0 出现。

**与 `review-anti-ai` 协作**：anti-ai 审计的禁忌词检查与本审计使用同一份 `prohibitions` 列表。两者的关系：
- `review-anti-ai` 把禁忌词计为 `error`（AI 味 + 写作质量问题）
- `review-sensitivity` 把禁忌词计为 `blocking error`（平台合规问题）
- 任一审计报错都必须修订

## 4. 内容边界

### 4.1 暴力程度

**判定依据**：题材预期 + 场景目的 + 描写详细度。

**类目参考**：
| 题材 | 暴力基线 | 触发阈值 |
|------|---------|---------|
| 玄幻/仙侠 | 战斗死亡 | 酷刑细节、对儿童暴力 |
| 都市/言情 | 冲突对峙 | 详细伤害过程、武器使用细节 |
| 悬疑/推理 | 命案现场 | 真实手法还原 |
| 儿童向 | 无暴力 | 任何显著暴力 |

`genre-config.json` 的 `genre` 字段决定基线。

### 4.2 成人内容

**判定依据**：是否构成独立成人桥段 + 是否有分级标注。

**审计步骤**：
1. 识别疑似成人桥段（场景、动作词、情感铺垫）
2. 判断是擦边还是独立桥段（独立桥段 = 可单独删除不影响主线）
3. 检查分级标注（`novel.json` 的 `contentRating` 字段）
4. 分级缺失 + 独立成人桥段 = blocking error

## 5. 平台合规性综合

**输出规则**：
- 任一平台禁忌词命中 → 不通过
- 任一 `prohibitions` 列表项出现 → 不通过
- 内容边界违规且未标注分级 → 不通过
- 全部 PASS → 通过

**重要**：本审计无 warning 级别。任何命中都直接 blocking。`error` 与 `pass` 是仅有的两个状态。
