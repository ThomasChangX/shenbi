# 跨章模式审计方法论

此参考由 `shenbi-review-long-span` 技能使用。提供 n-gram 比较算法、漂移公式、统计阈值参考。

> **重要：** 本文件描述**检测方法与公式**，不替代人工阅读判断。n-gram 高频 = 表达惯性信号，但需结合"是否影响阅读体验"判定。

## 1. 6 字 n-gram 重复率算法

### 1.1 提取

对每章文本做以下处理：

1. 去除标点与空白
2. 滑动窗口 6 个字符
3. 记录所有 n-gram 字符串及其位置

例：`"林轩看着他微微笑"` → `["林轩看着他", "轩看着他微", "看着他微笑", "着他微微笑"]`

### 1.2 比较

设：
- `A` = 本章 n-gram 集合
- `B` = 近 5 章 n-gram 集合（去重）

```
重复率 = |A ∩ B| / |A|
```

### 1.3 阈值参考

| 重复率 | 判定 |
|--------|------|
| < 0.10 | PASS（健康） |
| 0.10 - 0.15 | warning（需人工关注） |
| > 0.15 | error（高概率问题） |

阈值由 `genre-config.json` 的 `maxNgramRepetition` 覆盖。

### 1.4 误报过滤

以下情况不视为重复问题：
- 角色口头禅（已在 `voice_profile` 中标记）— 允许高频
- 专有名词（人名/地名/武功名）— 不计入
- 修辞性重复（章末回环、章首呼应）— 仅前 200 字 / 后 200 字内的回环允许

## 2. 段落长度漂移公式

设 `L_n` = 第 N 章平均段长（字）。

```
漂移率 = (L_n - L_{n-1}) / L_{n-1}
```

判定：
- 单章 `|漂移率| > 0.30` = warning
- 连续 3 章同向漂移且每次 > 0.20 = warning（趋势性问题）
- 方向交替 = 正常（节奏呼吸）

例：
```
Ch 10: 145 字 (基线)
Ch 11: 168 字 (+15.9%)
Ch 12: 195 字 (+16.1%)
Ch 13: 230 字 (+17.9%) ← 连续 3 章同向 + > 20% = warning
```

## 3. 句式开端检测

### 3.1 提取

提取每句前 6 字作"开端 n-gram"。统计每章中每个开端出现次数。

### 3.2 高频开端识别

按出现次数降序排列前 5 个开端。

判定：
- 单一开端 > 5 次 = warning
- 前 3 高频开端占总句数 > 40% = warning

### 3.3 过滤

- 短对话行（< 10 字）不参与统计
- 章首第一句（气氛建立）允许特殊开端

## 4. 意象循环检测

### 4.1 意象词来源

`genre-config.json` 的 `coreImages` 数组（作者标记的"核心意象"）。

例：`["剑", "月光", "血色", "桃花"]`

### 4.2 计数规则

- 本章：精确字符串匹配（不含标点）
- 跨章：累加 Ch N-2 至 Ch N

### 4.3 阈值

| 范围 | 阈值 | 判定 |
|------|------|------|
| 本章 | > 4 次 | warning |
| 跨 3 章累计 | > 8 次 | warning |
| 跨 5 章累计 | > 15 次 | warning |

> 例外：若该意象在 `coreImages` 中标记为 `intensify`（强调用），阈值 × 2。

## 5. 实施建议

n-gram 计算可用简单 Python 脚本：
```python
def ngrams(text, n=6):
    text = ''.join(c for c in text if c.isalnum())
    return [text[i:i+n] for i in range(len(text) - n + 1)]

def repetition_rate(current, historical):
    cur_set = set(ngrams(current))
    return len([g for g in cur_set if g in historical]) / max(len(cur_set), 1)
```

> **重要：** 算法只是信号源。最终判定必须结合人工阅读：n-gram 高频但读起来不重复 = 接受（数据噪声）；读起来重复但 n-gram 统计未达阈值 = 仍然需要修订（数据未捕捉到）。
